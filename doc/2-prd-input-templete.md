# 작성 완료 체크리스트

문서 작성이 완료되면 다음을 확인하세요:

## Phase 1: 제품 전략
- [ ] 제품 비전이 명확히 작성되었는가?
- [ ] 3-4개 페르소나가 구체적으로 정의되었는가?
- [ ] 사용자 여정 맵이 3개 이상 작성되었는가?
- [ ] 비즈니스 목표가 정량화되었는가? (3-5개 KPI)
- [ ] 성공 지표의 측정 방법이 명시되었는가?

## Phase 2: 제품 범위
- [ ] 도메인 정의서의 유스케이스가 사용자 스토리로 전환되었는가? (50개 이상)
- [ ] MoSCoW 우선순위가 명확히 정의되었는가?
- [ ] MVP 범위가 구체적으로 정의되었는가?
- [ ] 주요 기능별 상세 명세가 작성되었는가? (8-10개)
- [ ] Won't Have 항목이 명시되었는가?

## Phase 3: UX/UI
- [ ] 주요 화면 목록이 작성되었는가? (8-12개)
- [ ] 화면별 UI 요구사항이 명시되었는가?
- [ ] UX 원칙이 3-5개 정의되었는가?
- [ ] 접근성 요구사항이 포함되었는가?
- [ ] 주요 인터랙션 패턴이 정의되었는가?

## Phase 4: 릴리스 계획
- [ ] 3단계 릴리스 로드맵이 작성되었는가? (6개월)
- [ ] 각 Phase별 마일스톤이 정의되었는가?
- [ ] 주차별 계획이 구체적인가?
- [ ] 의존성 매트릭스가 작성되었는가?
- [ ] 리스크가 3-5개 이상 식별되었는가?
- [ ] 각 리스크에 대한 완화 전략이 있는가?

## Phase 5: 문서화
- [ ] 모든 섹션이 작성되었는가?
- [ ] 도메인 정의서와의 연계가 명시되었는가?
- [ ] 기술 설계 전환 가이드가 작성되었는가?
- [ ] 용어 사전이 일관되게 사용되었는가?
- [ ] 제품팀 리뷰가 완료되었는가?
- [ ] 최종 승인을 받았는가?

## 품질 체크
- [ ] 모든 목표가 정량화되었는가? ("빠르게" → "2초 이내")
- [ ] 모든 기능이 Must/Should/Could/Won't로 분류되었는가?
- [ ] 측정 불가능한 요구사항이 없는가?
- [ ] 모호한 표현이 구체화되었는가?
- [ ] 도메인 정의서와 일관성이 있는가?
