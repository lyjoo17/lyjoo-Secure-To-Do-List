# lyjoo-Secure-To-Do-List 실행 설명서

이 문서는 로컬 환경에서 lyjoo-Secure-To-Do-List 프로젝트를 실행하고 테스트하는 방법을 설명합니다.

## 1. 필수 요구사항

*   **Node.js**: v18 이상
*   **PostgreSQL**: v15 이상 (로컬 설치 또는 Supabase 등 클라우드 DB 사용)
*   **npm**: Node.js 설치 시 함께 설치됨

## 2. 데이터베이스 설정

1.  `.env` 파일 설정 (backend 디렉토리)
    *   `backend/.env` 파일이 존재하는지 확인하고, `DATABASE_URL`을 본인의 PostgreSQL 연결 문자열로 설정하세요.
    *   예시: `DATABASE_URL="postgresql://user:password@localhost:5432/mydb?schema=public"`

2.  데이터베이스 마이그레이션
    ```bash
    cd backend
    npx prisma migrate dev
    ```
    *   위 명령어를 실행하여 데이터베이스 테이블을 생성합니다.

3.  시드 데이터 생성 (선택 사항)
    ```bash
    npx prisma db seed
    ```
    *   테스트용 초기 데이터를 생성합니다.

## 3. 백엔드 서버 실행

1.  백엔드 디렉토리로 이동
    ```bash
    cd backend
    ```

2.  의존성 설치 (최초 1회)
    ```bash
    npm install
    ```

3.  서버 실행
    ```bash
    npm run dev
    ```
    *   서버가 `http://localhost:3000`에서 실행됩니다.

## 4. 프론트엔드 서버 실행

1.  새로운 터미널 창 열기

2.  프론트엔드 디렉토리로 이동
    ```bash
    cd frontend
    ```

3.  의존성 설치 (최초 1회)
    ```bash
    npm install
    ```

4.  서버 실행
    ```bash
    npm run dev
    ```
    *   브라우저가 자동으로 열리거나, `http://localhost:5173`으로 접속하여 애플리케이션을 사용할 수 있습니다.

## 5. 주요 기능 테스트 방법

### 회원가입 및 로그인
1.  `회원가입` 버튼을 클릭하여 새로운 계정을 생성합니다.
2.  생성한 계정으로 로그인을 진행합니다.

### 할 일 관리
1.  `+` 버튼을 클릭하여 새로운 할 일을 추가합니다.
2.  할 일 목록에서 체크박스를 클릭하여 완료 상태로 변경합니다. (폭죽 효과 확인!)
3.  할 일을 클릭하여 내용을 수정합니다.
4.  휴지통 아이콘을 클릭하여 할 일을 삭제합니다.

### 휴지통 및 복원
1.  사이드바의 `휴지통` 메뉴로 이동합니다.
2.  삭제된 할 일 목록을 확인합니다.
3.  복원 아이콘을 클릭하여 할 일을 다시 목록으로 되돌립니다.
4.  삭제 아이콘을 클릭하여 영구 삭제합니다.

## 6. 문제 해결

*   **DB 연결 오류**: `backend/.env` 파일의 `DATABASE_URL`이 올바른지 확인하고, PostgreSQL 서버가 실행 중인지 확인하세요.
*   **포트 충돌**: 3000번(백엔드) 또는 5173번(프론트엔드) 포트가 이미 사용 중인지 확인하세요.
